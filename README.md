# 学習のすゝめ ― トランザクション（Web 解説）

勉強を「気合と根性」で回すのをやめ、**始まり（トリガー）と終わり（報酬）ではさんだ一続きの流れ**を
**1件のトランザクション**として扱う勉強法を、生徒向けにインタラクティブに解説する Web 資料です。
原資料スライド「学習のすゝめ ― 不可分な一連の流れ」（Team ATLAS, CERN, 2018 ／ 小船幹生）に沿って作成しています。

「ディレクトリ構造ノート 見本」（`../ディレクトリ構造/webtool/`）と同じ作り（ルーズリーフ意匠・
濃紺見出し・白カード・折りたたみフォルダ・自己完結）を踏襲しています。数式は使わないため KaTeX は外し、
外部依存ゼロ（オフラインでそのまま開けます）。

## 収録内容
- **トランザクションとは？** … 概念（DBのトランザクション＝不可分な一続きの処理）＋3つのポイント
  ＋「引き金を引く」ドミノ・デモ（始動すると最後まで倒れる＝不可分を体感）
- **4つのトランザクション** … 実例をフォルダで開くと「トリガー → フェーズ → 報酬」の流れと
  時間の比例バーが出る（Tx1 60分／Tx2 60分／Tx3 土日200分／Tx4 本番520分）
- **週間プラン『教科配置』** … ロング↔ショート、通常版／現役・毎日登校版のトグル、7日の配置例
- **1日プラン『演習内容』** … ロング／ショートの過去問演習フェーズ（切替）
- **自分のトランザクションを組む** … トリガー・フェーズ・報酬を並べ、合計と可否をその場で判定するビルダー

## 使い方
- フォルダ（トランザクション）の行をクリックで開閉。フェーズのカード／時間バーをクリックで詳しい説明。
- 「すべて開く／閉じる」ボタン、週間・1日プランのトグル、ビルダーの入力はすべてブラウザ内で完結。
- キーボード操作（Tab 移動、Enter／Space で開閉）にも対応。

## ファイル構成
```
webtool/
├── index.html        … 構造＋エンジン（対話ロジック）
├── assets/
│   ├── data.js       … 内容（window.TX / WEEK / DAY / PRESETS）。編集はここ
│   └── tx.css        … スタイル（意匠はディレクトリ構造ノートを踏襲）
└── README.md
```

## 内容の編集
`assets/data.js` を編集します（エンジン `index.html` / `tx.css` は原則そのまま）。
トランザクションの追加は `window.TX` に1件足すだけ。合計時間は各フェーズの `min` の和と一致させます。

## 注意
- 教材名は原資料に載っている**例**です。
- 「12時間」「6時間」は**1日あたり**の目安（多い日は12時間／通学する日でも6時間確保）。週グリッドは
  「どの教科をどの曜日に置くか」の配置例で、週の総量ではありません。
- 原資料 p8（Tx4）は最終ノードのラベルが "Trigger" と誤記ですが、図柄は報酬袋・吹き出しは
  "スマホを見る" なので、ここでは報酬（Reward）として正しく扱っています。
